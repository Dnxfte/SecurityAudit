# SecurityAudit

Цей репозиторій демонструє приклад **безпечної конфігурації** для `JWT`, паролів і шифрування даних, а також містить кілька **неправильних конфігів** для порівняння.

## Основний правильний конфіг

Головний файл репозиторію: [secure_config.yaml](secure_config.yaml)

У ньому використано сучасні рекомендовані практики:

- `RS256` для JWT замість слабких або небезпечних варіантів
- зберігання ключів і секретів у змінних середовища
- перевірка підпису JWT та основних claims
- `Argon2id` для безпечного хешування паролів
- випадкова сіль і окремий `pepper`
- `AES-256-GCM` для сучасного автентифікованого шифрування
- короткий час життя токенів і базова політика для паролів

Також у файлі додані коментарі до кожного пункту, які пояснюють, чому саме це рішення вважається безпечним.

## Неправильні конфіги

Папка з прикладами: [Old configs](Old%20configs)

Ці файли додані **лише для порівняння**. Їх не можна використовувати як реальну конфігурацію.

- [crypto_config_we.yaml](Old%20configs/crypto_config_we.yaml)  
  Приклад явно небезпечної конфігурації: `algorithm: none`, вимкнена перевірка підпису, відсутня перевірка важливих claims, `SHA1` для паролів і слабший підхід до шифрування.

- [crypto_config_le.yaml](Old%20configs/crypto_config_le.yaml)  
  Приклад конфігурації з грубими помилками: `HS256` із жорстко прописаним секретом, фіксована сіль для паролів, `DES` і hardcoded ключ.

- [crypto_config_mi.yaml](Old%20configs/crypto_config_mi.yaml)  
  Приклад частково покращеної, але все ще неправильної конфігурації: `RS256` і `AES-GCM` уже кращі, але занадто слабкі параметри `Argon2`, а також використання `fixed_iv`, що є небезпечним для шифрування.

## Мета репозиторію

Мета цієї роботи не лише показати правильний конфіг, а й продемонструвати різницю між:

- безпечними та небезпечними алгоритмами
- правильним і неправильним зберіганням секретів
- сучасними та застарілими практиками захисту

## Висновок

[secure_config.yaml](secure_config.yaml) є основним і рекомендованим варіантом у цьому репозиторії.  
Папка [Old configs](Old%20configs) показує типові помилки, яких слід уникати під час налаштування безпеки.
